Problem
  • Manual regression testing was consuming 40–50% of sprint capacity — slowing feature delivery across 5+ apps
  • No standardised framework existed; each QA engineer worked independently with inconsistent coverage and documentation
My Approach
  • Scoped the problem: identified which apps had highest regression risk and defined automation priority order
  • Designed a modular Appium + PyTest + POM framework architecture — built for team onboarding, scalability, and CI/CD integration
  • Coordinated across 3 engineers to align on framework standards, assigned components, and managed delivery within 2 sprint cycles
  • Documented the full system — test strategy, architecture decisions, onboarding guide — so the team could maintain and extend it independently
Outcome
  • Regression cycle time cut by 40–50% — sprint capacity freed for feature development
  • 30%+ expansion in test coverage through risk-based prioritisation
  • Framework adopted as team standard; CI/CD-ready architecture enabled continuous quality monitoring
